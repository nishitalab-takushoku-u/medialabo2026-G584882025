「Webコンテンツ作成」の成果

本テーマについて

webページの基本的な記述方法であるHTML、それを装飾するCSS、動きのあるページを作るためのJavaScript、外部サーバと通信する方法について学習することで、webコンテンツ作成の基礎を学ぶ。

プロジェクト課題について

ラジオボタンに地名に対応したデータの検索キーを設定しておき、クリックされた時に取得してAjax通信のURLに入れることで地名に合った天気のデータを取得することができる。通信成功、通信失敗、通信の最後に実行する処理を記述する。通信に成功した場合、各データをli要素でletする。それをul要素の最後に差し込むことでデータを表示する。ラジオボタンの他の項目を選択した時にデータが累積して表示されるのを防ぐためにresulut要素を毎回の処理の最初に入れた。

数当てゲーム

実行する内容をhanteiに記述していく。まず、回数によって実行結果を変化させないといけないのでkaisuに1を足してテキストを更新する。その後、テキストボックスに入力された値を数値に変換してからyosoに代入する。ifで回数に応じて結果を3種類に分ける。kaisuが2以下なら、入力した値の正誤と誤りであれば入力した値より上か下か表示する。3の場合は正誤に加えて間違いの場合には正解を表示する。4以上であれば正誤に関わらず答えを表示する。ボタンを押した時にhanteiが実行されるようにquerySelectorでbuttonを指定し、イベントハンドラとしてhanteiを登録する。

演習の成果:課題4-1: 数当てゲームの作成

各授業会の学習事項

第１回 WWW と HTML

<>で囲まれた要素の役割、属性について学習した。HTML文書であることを宣言する<!DOCTYPE html>や、文字を強調する<strong>をはじめとした html,p,h1,head,body などの使い方を理解した。

演習の成果:演習1-4: HTMLのさまざまな要素

第２回 CSS による Web ページの装飾

CSSの記述を用いて要素それぞれの見た目を装飾する方法を学習した。head要素で外部CSSファイルを指定するとそのファイルの内容に基づいて記述が装飾される。CSSファイルには、〜要素を{〜に設定する}という風に記述する。例：p {color: green;}

演習の成果:演習2-3: ウェブブラウザ説明ページの装飾

第３回 JavaScript の基本

JavaScriptの概要とその基本的な構文について学習した。JavaScriptはJavaとは全く違う言語で、その例としてコンパイルがいらな言語である・変数の宣言において型名の宣言がいらないという点が挙げられる。for-of構文はfor (let n of ns)と記述することで、繰り返し i 回目ごとに n に ns[i] の内容そ代入することができる。

演習の成果:演習3-6: オブジェクトの参照

第４回 DOM と JavaScript

input要素でユーザ入力が実行できます。type属性を指定することで入力する値の種類を変更することができます。例として、textでテキストボックスが表示され、dateでカレンダーが表示されます。querySelectorでファイル中の指定した要素を検索します。イベントハンドラを記述、登録しておくことでイベントが発生した時にイベントハンドラに記述されたプログラムを実行することができる。

演習の成果:演習4-10: 足し算ページ

第５回 イベント処理

createElement構文を使うことで先にhtmlに要素を記述しておかなくてもjsが実行された時に新しく要素を追加することができる。要素を検索して、insertAdjacentElement構文で'beforeend’と記述すると検索した要素の内部に追加した要素を配置、'afterend'と記述することで検索した要素の直後に追加した要素を配置することができる。

演習の成果:演習5-7 DOMプログラミング(2)

第６回 プロジェクト課題を完成させよう

Ajaxはページ全体を再読み込みすることなく、ページの一部を更新することができます。sendRequestを使うことで指定されたURLのサーバと通信することができます。受け取ったデータはresp.dataで参照します。

工夫点，苦心点，強調点，感想

このテーマの学習目標通り、webページの基本的な記述方法であるHTML、それを装飾するCSS、動きのあるページを作るためのJavaScript、外部サーバと通信する方法について一通り学習できました。ですが、私はこのテーマの単元がとても苦手でとても苦労しました。もう一度各課題やプロジェクト課題をやってもできる気がしません。HTML、CSS、JavaScriptのはたらきについては理解できましたが各構文の使用方法が難しかったです。JavaScriptによるプログラムの記述、特にletを使った変数の宣言やその変数が持っている値、そして各種構文がどのように動作するのかが頭の中でこんがらがって実際にプログラムを組み立てる度にエラーが出てしまいました。例えば、let自体に型がないため代入するときに変数に今入っている型を気にしないといけないところ、短期間で多くの構文について学習したので構文のはたらきをパッと思い出すことができないところが特に苦手でした。エラーが出ても自分ではどこが間違っているか、どう修正すればいいかがさっぱりわからずに構文の意味を何度も何度も見返して質問しながら少しずつプログラムしていきました。今回はプロジェクト課題で天気予報を作ったので夏休み中に、NHKの番組表の課題も一人で作れるように練習しておきたいです。構文が苦手な分私はHTMLとCSSで、ページが見やすくなるようにレイアウトや色を工夫しました。授業で紹介された構文を一つ一つ試してみて、MDNのページで関連項目も確認して調整しました。具体的には、fieldsetを使ってタイトル付きの枠線で囲ったり、text-shadowを1pxずらしたものを8方向分（8個）作ることで文字にクッキリと輪郭をつけて背景色に関わらず文字が見やすくなるようにしました。イメージ通りの見た目にするために、ブラウザで何度も確認しながら微調整を重ねて、細部までこだわって作りました。全体を通して、JavaScriptの構文の難しさがわかりました。エラーが発生するたびに原因を調べ、思い通りにページが動いた瞬間には大きな達成感を感じました。今回のテーマを通じてWebコンテンツ作成の奥深さと楽しさを知ることができたため、今後はJavaScriptの理解をさらに深めて、より高度で動きのあるWebページを作ることができるよう学習を続けていきたいです。
